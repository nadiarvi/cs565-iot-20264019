# CS565 IoT Data Science Mini Project

This repository contains the final submission notebook:

- `20264019.ipynb`

The notebook is intended to run in Google Colab. It installs its Python package dependencies inside the notebook with the existing `!pip install` cell, so this submission does not include a separate `requirements.txt`.

## How to Run

1. Open `20264019.ipynb` in Google Colab.
2. Select a GPU runtime: `Runtime` -> `Change runtime type` -> `GPU`.
3. Place the course-provided data file in Google Drive as described below.
4. Run the notebook cells from top to bottom.

The required course-provided data file is:

```text
features_stress_fixed_K-EmoPhone.pkl
```

This data file is not committed to the repository.

## Dataset Placement

Recommended Google Drive folder:

```text
MyDrive/KAIST (Graduate)/CS565 IoT Data Science/Mini Project/Models and Codes/
```

Place the data file directly inside that folder:

```text
MyDrive/KAIST (Graduate)/CS565 IoT Data Science/Mini Project/Models and Codes/features_stress_fixed_K-EmoPhone.pkl
```

```python
ROOT_DIR = Path("/content/drive/MyDrive/KAIST (Graduate)/CS565 IoT Data Science/Mini Project/Models and Codes")
```

If you place the dataset somewhere else, update `ROOT_DIR` and/or `DATA_PATH` in Section 0 before running the loading cells. The same Section 0 path adjustment applies if you run the notebook with another compatible preprocessed data file.

## Generated Outputs

The notebook writes generated artifacts under:

```text
MyDrive/KAIST (Graduate)/CS565 IoT Data Science/Mini Project/Models and Codes/Output/
```

Section 0 creates and controls the main output directories:

- `Output/checkpoints/`: model weights and preprocessing masks.
- `Output/results/`: JSON result files for trained models.
- `Output/hpo/`: hyperparameter optimization trial files and HPO JSON results.
- `Output/cache/`: cached arrays for sequence and split construction.
- `Output/figures/`: saved plots generated by the visualization cells.

Section 0 also controls `SEED` and the `RUN_*` toggles used to enable or skip expensive experiment sections.

## Notebook Structure and Code Quality

The notebook is organized to match the submission's code quality goals:

- Metadata: title, author identifier, project context, and final attribution are included in the notebook.
- Reproducibility: all paths, seeds, experiment constants, and run toggles are centralized in Section 0.
- Modularity: preprocessing, cross-validation, model training, loading, summarization, and plotting are implemented as reusable functions.
- Readability: sections are ordered from setup and data loading through model experiments, HPO, results, visualizations, and references.
- Visualization: final comparison and ablation plots are generated and saved under `Output/figures/`.
- References: related work and implementation references are listed in the notebook.
- Borrowed-code acknowledgment: the course-provided preprocessed K-EmoPhone feature file is acknowledged, while model design, training, evaluation, and analysis code are identified as original work.
